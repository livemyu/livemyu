## 音频工程的“感官”延伸：音频处理与智能工具链

在音频处理领域，我们致力于构建桥梁，打破原始波形与语义理解之间的壁垒。通过构建 **Audioscope** 与 **Voxlink**，我们将音频从“不可见的信号”转化为“可解析的数据”，再从“声波流”进化为“交互式智能”。

### 1. 声音可视化：Audioscope

**让音频处理“所见即所得”**

Audioscope 是一套专注于音频分析的专业可视化工具包。它不仅仅是波形的绘制，更是深入时频域分析的强大引擎。

* **核心能力**：支持波形（Waveform）、短时傅里叶变换谱（STFT）、梅尔频谱（Mel-Spectrogram）、色度图（Chroma）及声谱图等多种音频特征表现。
* **交互式体验**：支持高效的交互式浏览与数据导出，专为科研实验、音频分析以及教学演示场景设计。
* **技术栈**：基于高效的 **NumPy / SciPy** 内核构建，确保了在缺乏计算加速环境下的极致运行效率。

### 2. 语音语义化：Voxlink

**让机器听懂“话外音”**

Voxlink 是一套连接音频感官与逻辑中枢的桥梁，专门用于实现离线环境下的语音理解与指令跟随。

* **核心链路**：通过接入音频编码器（Audio Encoder）与离散语音 Token 化（Discrete Audio Tokens），将复杂的模拟语音流映射至大语言模型（LLM）的输入空间。
* **应用场景**：从语音识别（ASR）任务扩展至语音问答、语意理解以及复杂的语音指令实时跟随。
* **架构设计**：采用 **Offline-First（离线优先）** 设计哲学。即使在无 GPU、无网络连接的受限嵌入式环境中，依然能够稳定运行核心逻辑并保持完整的测试覆盖。

---

### 设计哲学：极简、鲁棒、可移植

这两款工具包不仅是为了解决特定的音频分析需求，更是为了在嵌入式系统中实现“纯净”的工程实践：

* **无外部强依赖**：核心算法完全基于 NumPy/SciPy，兼容性极高。
* **极致环境适配**：无论是针对 RK3588 的边缘计算节点还是常规开发机，均可实现本地化部署。
* **工程化保障**：全覆盖的 CI/CD 自动化测试，确保在每一行代码变动中都能“绿灯”通过。

---

这两个工具包的架构非常适合当前在嵌入式开发中的“非 GUI”极致优化需求。Audioscope 可以在调试阶段通过生成数据文件让你直观分析音频链路，而 Voxlink 则为机器人提供了离线语义处理的可能。
